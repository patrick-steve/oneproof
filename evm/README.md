# evm/ — Tier 3 stretch: EVM ↔ Stellar Groth16 portability

> implement.md §6 portability gesture. Same BN254 Groth16 proof verifies on:
>
> - **Stellar** (already done, Tier 1): contract
>   `CDXK7XNCTAGNT73NNQ6POAOSTP2KZVQ5IW6ZWL25ZEYNMRTARBKJZQUO`,
>   single-proof verify_one cost ≈ 30,556 stroops on testnet.
> - **EVM** (this directory): Solidity verifier auto-generated by snarkjs
>   from the same `circuits/groth16_batch/target/circuit_final.zkey`. We
>   measure on-chain gas via Foundry's `forge test --gas-report`.

The point isn't that EVM verification is cheaper or more expensive than
Stellar — it's that the SAME proof bytes work unchanged. BN254 Groth16
is curve-portable because both EVM (via the precompiles ecrecover-style
pairing op at address 0x08) and Stellar Protocol 25/26 (via
`soroban_sdk::crypto::bn254`) implement the standard pairing equation
over the same curve.

## Run

Prereq: `foundryup` installed.

```
./scripts/run-evm-verify.sh
```

The script regenerates `src/Verifier.sol` and `test/Verifier.t.sol`
from the current Tier 1 artifacts so it's always in sync with the
on-Stellar verifier.
